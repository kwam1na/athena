Summary

The plan introduces a neutral operationAdmission rail while keeping Convex public exports in their normal direct mutation({ ... }) shape. Shared demo becomes the first adapter-backed non-human consumer, with operations/openWorkInventoryReviews as the proving path before demo-reachable writes and migration controls for remaining public writes.

Scope guardrail: public write mutation admission is in scope. Public reads, public actions, role-model redesign, a policy DSL, and broad provider migration are deferred.

Key Decisions

  • Keep Convex exports direct and wrap handler execution, not file/export registration. Raw means a public write export without admission.
  • Create packages/athena-webapp/convex/operationAdmission/ for neutral contracts and structural checks.
  • Move the Athena-wide capability catalog out of sharedDemo into a neutral platform-owned module.
  • Resolve actor kind before normal-user auth so recognized demo principals never fall through.
  • Require resource-owner scope resolvers instead of trusting caller-supplied store ids.
  • Require expected restore epochs for browser-originated shared-demo writes.
  • Use AST/source-derived public export discovery plus operation definitions and shrinking legacy exemptions.

Implementation Units

U1. Neutral contract

Define operation definitions, actor/scope/readiness/effect contracts, provenance, and handler wrapping.

U2. Actor adapters

Implement normal-user and shared-demo adapters with explicit non-fallback shared-demo behavior.

U3. Structural coverage

Add AST-backed public export checks, operation definition validation, and migration exemptions.

U4. Proving mutation

Migrate operations/openWorkInventoryReviews under daily_operations.write, including the Operations queue caller and expected demo restore epoch.

U5. POS writes

Migrate POS sale, quick-add, register, and sync write boundaries without broad POS refactors.

U6. Ops and cash

Migrate operations, cash-control, stock-adjustment, approval, staff-auth, and staff-message demo writes.

U7. Storefront effects

Declare storefront fulfillment and protected-effect boundaries without provider dispatch migration.

U8. Migration controls

Enforce no-new-raw public writes and track remaining waves with owner/reason exemptions.

U9. Legacy cleanup

Retire helper-only shared-demo admission where migrated groups no longer depend on it and update docs.

Verification Strategy

  • Pure unit tests for operation definitions, adapter decisions, and handler non-invocation on denial.
  • Static AST coverage for public write mutation declarations, missing metadata, raw exports, and legacy exemptions.
  • Focused Convex/Vitest coverage for normal user, unauthenticated caller, valid demo, expired demo, wrong store, stale epoch, and restore-in-progress paths.
  • Return-contract proof with existing command-result validators for migrated public operations.
  • Merge readiness remains the repo-owned bun run pr:athena gate during implementation.
