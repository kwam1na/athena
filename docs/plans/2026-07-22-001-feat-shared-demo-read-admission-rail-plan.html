Shared Demo Read Admission Rail

Type: feat Status: active Date: 2026-07-22 Source: markdown plan

Summary. Build a query-side counterpart to the operations admission rail for shared-demo reads, prove it first on Operations viewing, and leave a checker-backed inventory for follow-up onboarding waves.

Scope

Must Land

  • Read admission definitions and wrapper for public Convex queries.
  • Shared-demo read adapter with server-owned store/org clamp.
  • Operations viewing proof via Daily Operations snapshot reads.
  • Complete checker-backed inventory for every non-reporting ad hoc shared-demo read surface.

Excluded

  • No migration under packages/athena-webapp/convex/reporting/** or reports.read bridges.
  • No generic auth expansion for write-shaped read capabilities.
  • No change to write admission, protected effects, or demo history fixtures.

Key Decisions

  • Proving path is Operations viewing, not Open Work.
  • Open Work follows as the first tracked onboarding wave after the first PR.
  • Read semantics remain separate from write readiness and effects.
  • Expired or wrong-store shared-demo reads are recognized denials with no normal-auth fallback.
  • Reporting reads remain unmanaged by this migration.

Implementation Units

U1. Inventory and Checker Foundation

Create exact non-reporting read inventory and a checker that distinguishes legacy, migrated, and untracked query states.

U2. Read Admission Rail Core

Add query wrapper, read definitions, context injection, and shared-demo read adapter tests.

U3. Operations Viewing Proving Path

Migrate Daily Operations snapshot reads off write-shaped demo capabilities and prove current-day shared-demo viewing.

U4. Open Work and Operations Queue

Migrate queue and count reads in a follow-up after the Operations viewing proof.

U5. Remaining Read Waves

Onboard shell, catalog, stock, POS, evidence, storefront, and telemetry reads in bounded follow-up waves.

U6. Docs and Delivery Proof

Document the pattern, rebuild graphify, and include delivery evidence for the PR.

Verification Strategy

  • Focused tests for read wrapper sequencing, shared-demo admission, wrong-store denial, expired admission, and normal-user fallback.
  • Daily Operations exported-query tests proving Operations viewing uses read admission at exported query boundaries.
  • Read checker tests proving definition plus wrapper coverage and reporting path/capability exclusion.
  • Repo-level validation with bun run pr:athena before the PR is marked mergeable.

Risks

Main risk is accidental shared-demo overreach. The mitigation is to make every migrated read declare scope, test wrong-store denial, keep reporting excluded, and prevent recognized demo actors from falling through to normal auth.
