Preserve Carry-Forward Work During EOD Auto-Completion

Extend EOD auto-completion so Athena can close blocker-free EOD Reviews with supported review evidence and carry-forward work, while preserving unresolved work for Opening Handoff.

Type: feat Status: active Date: 2026-06-26 Source: Markdown plan

Scope

Core shift: carry-forward is no longer a hard EOD auto-completion blocker by itself. Athena may complete the EOD Review when blockers are zero and review policy passes, but carry-forward remains unresolved and visible for Opening.

  • Keep blockers strict.
  • Keep supported review categories limited to cash_variance and voided_sale.
  • Preserve carry-forward work item IDs, counts, and snapshot evidence.
  • Do not create new carry-forward work items from automation.
  • Do not treat carry-forward as policy-reviewed review evidence.

Key Decisions

Decision Rationale
Replace carry-forward skip with preserved evidence. Carry-forward is already durable handoff work and should not block closing when the day has zero blockers.
Keep review and carry-forward semantics separate. policyReviewedItemKeys should only mean review evidence checked by policy, not work resolution.
Let command-time re-read win. Daily Close is a store-day command boundary; the final server snapshot and policy check prevent races from closing blocked or over-threshold days.
Preserve IDs, stored snapshot evidence, and applied run evidence. Opening Handoff needs durable IDs, history needs immutable report evidence, and the automation ledger must match the completed close.

Critical Test Themes

  • Clean day plus carry-forward completes when clean-day policy is enabled.
  • Low-risk review plus carry-forward completes inside thresholds.
  • Unsupported review, mixed unsupported review, blockers, and threshold failures still skip.
  • Automation completion stores carry-forward IDs, counts, snapshot items, and Athena attribution.
  • Applied automation evidence matches the command-time completed close/report snapshot.
  • Wrong-store, wrong-organization, terminal, duplicate, missing, or unmappable carry-forward items fail closed.
  • Broad readers see safe attribution and carry-forward shells/counts but not restricted financial or review metadata.
  • Opening Handoff surfaces carry-forward work from an Athena-completed prior close.
  • Dry-run, skipped, redacted, no-carry-forward, and carry-forward-present copy states stay distinct.

Risks

Risk Mitigation
Automation appears to resolve carry-forward work. Keep copy and data model explicit: preserved for Opening, not resolved.
Opening loses carry-forward handoff. Persist carryForwardWorkItemIds and test Opening context.
Restricted evidence leaks. Keep redaction boundaries and add broad-reader tests.
A race closes a newly blocked day. Command-time snapshot re-read wins and skips if blockers or unsupported review appear.
