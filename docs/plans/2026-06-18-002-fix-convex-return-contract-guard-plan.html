fix: Guard Convex return validator contract drift

Type: fix · Status: active · Date: 2026-06-18 · Source: markdown plan
Summary Requirements Implementation Units Risks Verification

Summary

Add a reusable Convex return-validator conformance helper and a repo-level inferential-review guard so changed public Convex functions with explicit returns validators must carry executable contract proof, not just loose exported-validator string checks.

Audit finding: PR #522 changed a public Convex response shape, while PRs #523 and #524 were needed to patch missed return-validator fields after production rejected valid handler results.

Requirements

  • Target the general Convex ReturnsValidationError class, not terminal-health-specific fields.
  • Validate representative returned values against exported Convex return validators.
  • Block changed public Convex functions with explicit returns validators when no executable contract proof changes with them.
  • Keep public Convex modules thin and application-owned behavior intact.
  • Keep Athena generated docs, Graphify, and pr:athena validation aligned.

Key Decisions

  • Use a serialized-validator assertion helper instead of expanding string-token checks.
  • Use scripts/harness-inferential-review.ts as the repo-level enforcement point because it already gates pr:athena.
  • Require changed proof near the public Convex change so reviewers see the contract was considered in that PR.

Implementation Units

U1. Reusable return-validator conformance helper

Create packages/athena-webapp/convex/lib/returnValidatorContract.ts and tests for nested objects, unions, arrays, records, ids, primitives, optional fields, missing fields, and extra returned fields.

U2. Generic inferential-review guard

Extend scripts/harness-inferential-review.ts and its tests so changed public Convex functions with returns: require changed executable contract proof.

U3. Representative public-contract fixture

Use terminal health as one real regression fixture for the generic helper, proving the helper would have caught the #522 and partial #523 misses.

U4. Harness documentation and generated artifacts

Update harness registry guidance, regenerate docs, and rebuild Graphify.
