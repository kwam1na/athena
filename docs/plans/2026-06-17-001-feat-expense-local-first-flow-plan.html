feat: Align expense flow to local-first POS

Problem Frame

Expense product search already uses local POS catalog behavior, but expense still waits on Convex mutations for the durable cart/session transaction path. This makes trusted inventory slower than exception rows and less resilient than POS.

Planning assumptions: trusted zero-available physical items proceed locally and route conflicts to review; expense gets first-class local events; expense sync is drawerless and terminal-scoped; sessions do not expire in the cashier path; cloud inventory effects happen during projection, not local cart edits.

Requirements

  • Expense actions are durable locally before cashier acceptance.
  • Local state survives reloads, held sessions, voids, completion, and offline/online transitions.
  • Sync is ordered and idempotent with stable local-to-cloud mappings.
  • Trusted, provisional, and pending checkout SKU sources stay distinct.
  • Local receipt/report evidence exists before cloud mapping.
  • Expense remains drawerless and does not reuse POS sale commands.
  • Expense sessions remain recoverable until explicitly completed, held, voided, or reconciled.

Key Decisions

First-class expense events
Clear projection and audit semantics without overloading POS sales.
Local append before success
Convex latency no longer gates cashier acceptance.
No cloud cart holds
Trusted stock changes apply once during projected completion.
Source-aware identity
Same-SKU trusted, pending, and provisional rows do not collapse incorrectly.
Drawerless sync scope
Expense events do not fake POS register-session identity.
No local expiry gate
Session age becomes sync/review metadata, not a cashier blocker.

Implementation Units

U6. Create a drawerless expense sync scope

Add an explicit sync scope for expense events so upload cursors, mappings, and conflicts do not depend on POS register-session ids.

U1. Define expense local event contract and storage projection

Add expense event types, local ids, ordering, and a read model for active, held, pre-completion voided, completed, synced, and needs-review sessions.

U2. Move expense UI/session operations to local-first commands

Make add/update/remove/clear, create/hold/resume/void, and complete use local appends and local read state.

U3. Project local expense events into Convex idempotently

Upload expense local events through the sync boundary and map them into existing expense session/transaction records once.

U4. Handle trusted inventory conflicts and exception sources during projection

Apply trusted stock effects once, skip trusted effects for pending/provisional rows, and create review evidence for conflicts.

U7. Provide local receipt/report handoff before cloud mapping

Let locally completed expenses display, print, reload, and open report detail before a cloud transaction id exists.

U5. Expose synced expense status, cloud report mapping, and docs

Connect cloud-mapped expense reports back to local identifiers, expose synced/needs-review/failed-sync status, and document the new boundary and validation coverage.

System Impact

  • Expense UI shifts from Convex mutation-first to local command/read-model-first.
  • Convex remains the cloud reporting and reconciliation target.
  • Projection failures become sync/review state instead of cashier rollback.
  • POS sale flow, drawer gates, and payment flows stay unchanged.
  • Expense sync uses terminal/session scope rather than synthetic register-session ids.

Delivery Sequence

Track as a coordinated Linear batch: U6 -> U1 -> U2 and U3 where safe -> U4, with U7 after U2 and U5 after U4 plus U7. Land through one integration PR, loop relevant reviewer agents to unanimous approval, squash-merge to remote main, fast-forward the local root checkout, then deploy Convex and Athena webapp surfaces from clean local main.

Verification Strategy

Use test-first implementation for new behavior. Focused sensors should cover expense local command/read-model behavior, sync ingestion/projection idempotency, trusted zero-available inventory review, source metadata preservation, and receipt/report handoff. Finish with changed-file lint, Convex validation, TypeScript, build, graphify rebuild, generated artifact refresh, and the repo harness review.
