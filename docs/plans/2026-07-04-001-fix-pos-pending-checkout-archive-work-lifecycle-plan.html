Fix plan · 2026-07-04

Retire archived pending checkout review work

Move POS pending checkout review work lifecycle out of read-time hiding and into source-owned archive/unarchive behavior, so Open Work and EOD Review agree on what is still actionable.

Problem

Archived provisional POS pending checkout products leave associated pos_pending_checkout_item_review operational work items open. Open Work filters those rows out because the product is archived, while EOD Review reads open operational work directly and still carries them forward.

SurfaceCurrent behavior
Open WorkShows the synced sale inventory reviews because archived provisional pending checkout rows are hidden at queue projection.
EOD ReviewShows stale pending checkout review rows because their work item status is still open.

Direction

  • Archive cancels every matching open or in_progress pending checkout review work item and clears the source pointer.
  • The pending checkout item stays pending_review or flagged; archive is not a review decision.
  • Unarchive creates one fresh open review work item when the source still needs review and no current open review exists.
  • Resolved sources do not recreate review work.
  • A mandatory dry-run and bounded production repair retire existing stale open rows with the same helper semantics.
  • Live product availability mutations abort if duplicate-work scans are incomplete, so product state and work lifecycle cannot commit partially.

Key Decisions

DecisionRationale
Use cancelled for archived workArchive makes review unactionable; it does not mean manager review completed.
Keep source status unchangedThe product is still a POS pending checkout item and can be reviewed after unarchive.
Treat operationalWorkItemId as current pointerArchive clears the stale pointer; unarchive points it at the fresh actionable row.
Create a fresh work item on unarchivePreserves retired history and avoids resurrecting stale metadata.
Route all availability transitions through the helperarchive, unarchive, and generic update must agree.
Abort on incomplete live scansIf the helper cannot prove the full current work set, the source product mutation does not commit.
Repair production dataCode fixes prevent new drift; existing stale rows still need bounded dry-run and repair.

Acceptance

  • Archived provisional pending checkout review work is terminal and absent from both Open Work and EOD carry-forward.
  • Unarchived actionable pending checkout sources get exactly one fresh open review work item.
  • Resolved pending checkout sources do not recreate review work.
  • Dry-run and mutating repair paths identify and retire current stale rows safely with candidate ids, skip reasons, cursors, and idempotent retry behavior.
  • Incomplete duplicate scans abort archive, unarchive, and generic update availability transitions.
  • Focused Convex tests pass for product lifecycle, queue projection, and EOD carry-forward.
