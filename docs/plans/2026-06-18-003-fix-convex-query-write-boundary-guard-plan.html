Plan

fix: Guard Convex query write-boundary drift

Date: 2026-06-18 · Status: active · Type: fix

Summary

Add a type-level and harness-level guard so Convex query handlers cannot obtain mutation-only DB capabilities through shared repositories or services. The immediate terminal-recovery hotfix removed the read-path write; this plan closes the broader production gap.

Problem Frame

Production hit t.db.patch is not a function from a repository method while serving listTerminalRecoveryCommands, a public Convex query. The query passed QueryCtx into a write-capable repository factory that accepted QueryCtx | MutationCtx and cast to MutationCtx inside write methods.

Requirements

  • Keep terminal-recovery listing read-only and filter expired commands without persistence.
  • Split terminal-recovery repository capability into read-only and mutation-capable factories.
  • Split query-safe listing behavior from mutation-side expiry and command writes.
  • Add incident regression tests using query-shaped db objects without write APIs.
  • Add a deterministic harness:inferential-review guard for changed Convex query/write-boundary violations, including alias/cast fixtures and changed query-facing service or repository paths.
  • Update generated validation guidance and rebuild Graphify.

Implementation Units

U1. Terminal Recovery Repository Capability Split

Introduce a read-only repository interface and factory for query paths, keep write methods behind a MutationCtx-only factory, and update public terminal recovery query/mutation call sites accordingly.

U2. Public Query Regression Coverage

Add service, repository, and public module tests proving terminal recovery listing does not call write methods and works with a query-shaped context.

U3. Deterministic Inferential-Review Guard

Fail changed Convex query files that directly call mutation-only DB APIs, and fail changed query-facing repositories or services that mix QueryCtx | MutationCtx with MutationCtx casts and write calls.

U4. Harness Guidance, Generated Docs, and Learning

Document the new rule through the harness registry, regenerate package guidance, rebuild Graphify, and add a solution learning for the production bug class.
