Configure POS store-day auto-start with manager review

Canonical source: 2026-06-11-001-feat-pos-store-day-auto-start-settings-plan.md

Summary Decisions Design Implementation Units Risks Verification

Summary

Add a full-admin POS settings control for store-day auto-start. At the configured local time, Athena starts Opening Handoff even when blockers exist, then preserves those blockers as manager-review evidence so cashier POS access is not held closed.

Cashier continuity

POS unlocks through the normal started store-day record, not a cashier-side bypass.

Manager review

Opening blockers, review items, and carry-forward items remain visible after Athena starts the day.

Strict manual path

Human starts still reject hard blockers and require acknowledgements.

Requirements

  • Full admins can configure Opening auto-start from POS settings.
  • The setting stores local start time and timezone offset for scheduled automation.
  • Configured automation starts the store day even when Opening has blockers.
  • Blocker and review evidence stays durable for manager follow-up.
  • Manual start-day and EOD rules stay unchanged.

Key Decisions

  • Extend the existing opening.auto_start action instead of adding a new action.
  • Add explicit policy fields for local start time and blocker handling.
  • Persist review evidence at the automation opening boundary.
  • Place configuration in POS settings but gate it to full admins.
  • Keep drawer, register session, opening float, and Cash Controls ownership unchanged.

Technical Flow

POS SettingsFull admin configures auto-start and local time.
Policyopening.auto_start stores mode, time, timezone, and blocker handling.
CronHourly automation checks whether local start time has arrived.
Opening SnapshotAthena reads current blockers, review items, and carry-forward work.
StartAutomation starts the store day when policy allows review-required starts.
EvidenceAutomation run, opening record, and event preserve manager-review context.
