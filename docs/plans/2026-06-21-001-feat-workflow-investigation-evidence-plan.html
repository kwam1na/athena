Summary. Extend Athena's POS workflow-trace foundation into stock receiving, service cases, online orders, online returns/refunds/exchanges, and scheduled jobs. The work reuses existing trace infrastructure, keeps source ledgers authoritative, and adds investigation evidence without changing frontline workflow outcomes.
Requirements Key Decisions Implementation Units System-Wide Impact Risks Validation

Requirements Digest

  • Expose investigation evidence from relevant stock, service, online order, online return/refund/exchange, and scheduled-job surfaces.
  • Reuse the existing workflow trace core, presentation query, route, and link component when a durable lifecycle trace is appropriate.
  • Keep operationalEvent, inventory movement, SKU activity, payment allocation, automationRun, and source records as facts of record.
  • Make evidence writes best-effort and non-blocking for business mutations.
  • Add replay-safe persisted event-key dedupe for retry-prone milestones.
  • Enforce source-surface permission parity and minimize sensitive trace/run evidence.
  • Use explicit per-surface placement and state rules for trace links and scheduled-run summaries.
  • Avoid a new command center, recommendation engine, or blocking review workflow.

Implementation Units

U1 Shared Evidence Foundation

Add optional persisted event-key dedupe, best-effort writer convention, adapter tests, source-permission trace read authorization, evidence minimization, and backward-compatible presentation behavior.

U2 Stock Receiving

Add PO-centered lifecycle traces with receiving batch, SKU, inventory movement, vendor, and work-item refs.

U3 Online Orders

Add order lifecycle traces for creation, payment, readiness, fulfillment, cancellation, and external references.

U4 Returns, Refunds, Exchanges

Add linked subflow evidence for approvals, refund reservation/finalization/release, restock, and replacement branches.

U5 Service Cases

Add service-case traces across intake, appointment conversion, approvals, payment, inventory use, completion, and cancel states.

U6 Scheduled Runs

Add store-partitioned, authorized run evidence for applied, skipped, rerun, and partial-failure cron outcomes while preserving automationRun semantics.

U7 Surface Links

Render trace/run links in explicit header, row action, metadata, and Daily Operations evidence placements with defined empty, loading, permission, unavailable, partial-failure, and zero-candidate states.

U8 Docs and Validation

Capture the reusable pattern, refresh generated artifacts, rebuild Graphify, and run the Athena proof ladder.

System-Wide Impact

  • Data integrity: traces summarize and link; ledgers remain authoritative.
  • Idempotency: receiving submission keys, status transitions, refund retries, and scheduled-run keys need duplicate protection, including cron reruns for the same scheduled window.
  • Authorization: trace reads by id and lookup must assert store/org and source-domain permission before exposing broader customer, payment, service, and automation evidence.
  • Trace access contract: U1 must provide an executable access descriptor or workflow-type authorizer path before sensitive domain traces ship.
  • Operational copy: partial failures use normalized, operator-safe wording, and trace/run details avoid raw provider payloads, payment tokens, customer contact data, and service notes.
  • Generated artifacts: schema/client changes may require Convex codegen, harness docs, and Graphify rebuild.

Risks and Mitigations

RiskMitigation

  • Domain tests for stock receiving, service cases, online orders, return/refund/exchange, and scheduled jobs.
  • Component tests for trace-link rendering in changed operator surfaces.
  • Convex return-validator checks where public contracts change.
  • Production rollout deploys Convex first, then the Athena webapp surface.
