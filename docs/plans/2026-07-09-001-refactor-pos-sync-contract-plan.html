Summary

Make shared/posLocalSyncContract.ts the authoritative source for POS local-sync event identity, upload metadata, literal vocabulary, and payload types. Browser upload conversion and Convex ingestion should consume that same contract metadata while Convex-local helpers build real v.* validators, preserving current local-first sync semantics, server-authoritative rejection, and projection behavior.

Core guardrail: browser payload conversion stays best-effort. Server ingestion remains responsible for semantic rejection and retry-visible outcomes.

Requirements

  • R1-R3: Browser upload and Convex ingestion use one canonical source for event identity, upload metadata, literal vocabulary, and dot-to-underscore mapping.
  • R4-R5: Convex public argument validators are composed from contract-owned metadata while remaining real Convex validators, and Convex parsed types derive from shared payloads where possible.
  • R6-R8: Existing sync semantics, projection behavior, POS/expense cursor identity, and retry/idempotency stay stable. Retry lookup remains store + terminal + localEventId before cursor scope.
  • R9-R10: register_reopened remains server/review-supported only, and activity events remain outside core sync.
  • R11-R12: Characterization coverage, focused POS sync validation, generated-artifact handling, Graphify, and PR gate coverage prove the refactor.
  • R13-R14: Raw staff proof and terminal sync secret values stay transient, and public auth/role/terminal-secret gates remain before ingestion.

Key Decisions

Contract-Owned Event Map

The shared contract owns upload event type, local dot-event source, sync scope, browser upload eligibility, and intentional exceptions.

Convex-Local Validators

Convex validator construction consumes shared metadata from a Convex-side helper so shared browser-safe code never imports Convex server modules.

Server-Authoritative Rejection

Browser conversion continues normalizing and redacting. Ingestion keeps semantic validation for totals, references, cursor identity, and retry mismatch.

Activity Stays Separate

Register-session activity reporting remains adjacent to, but outside, core POS local sync.

Implementation Units

U1. Characterize Contract Parity and Failure Boundaries

Add exhaustive mapping and failure-boundary tests for every PosLocalEventType, including core-upload, activity-only, local-only, server/review-only, unknown event rejection, public validator rejection, POS/expense identity, auth/secret gates, raw proof/secret omission, and log redaction.

Retry invariant: ingestion retry lookup is keyed by store, terminal, and localEventId before cursor scope. If same-terminal POS/expense collisions exist, execution stops for remediation before refactoring.

Primary files: syncContract.test.ts, posLocalStore.test.ts, convex/pos/public/sync.test.ts, ingestLocalEvents.test.ts.

U2. Promote Shared Contract Metadata and Browser Consumers

Add browser-safe event metadata and helper exports to the shared contract, then replace browser uploadability and mapping duplication while preserving normalization behavior and keeping staffProofToken transient.

Primary files: shared/posLocalSyncContract.ts, posLocalStore.ts, syncContract.ts, local sync tests.

U3. Replace Convex Public Validators With Contract-Derived Validators

Create a Convex-side validator helper that consumes shared metadata and powers strict public upload validators. Share stored schema event/status literals only where stored row compatibility stays unchanged.

Primary files: convex/pos/application/sync/posLocalSyncValidators.ts, convex/pos/public/sync.ts, convex/schemas/pos/posLocalSyncEvent.ts.

U4. Derive Ingestion Types and Add Explicit Parser Guard

Alias Convex payload types to shared payloads where possible, keep semantic validation in ingestion, reject unknown events through exhaustive dispatch, and persist only staff proof hashes.

Primary files: convex/pos/application/sync/types.ts, ingestLocalEvents.ts, sync and projection tests.

U5. Document, Validate, and Prepare Delivery Handoff

Update durable solution guidance, refresh generated and Graphify artifacts when needed, then validate with focused POS sync tests and the full Athena PR gate.

Risks

Risk Mitigation

Shared module imports Convex server code Keep Convex imports in Convex-side helper modules only.
Public and stored validator shapes are conflated Share only event/status literals unless tests prove stored row compatibility.
Browser strict validation changes retry/review behavior Preserve browser best-effort normalization and server-authoritative rejection.
Unknown internal event falls through as register_reopened Add explicit known-event guard and ingestion regression coverage.
Raw proof or secret values leak through refactor Keep raw values transient, persist only hashes, and test returned errors/conflicts/mappings/logging omit raw values.
Public auth or terminal-secret checks are bypassed Preserve existing public sync gate order and regression-test no ingestion call on auth or secret failure.
Runtime surfaces are changed but not deployed Current finish line intentionally stops at merged PR and local root alignment; report production deploy as skipped by user instruction.
