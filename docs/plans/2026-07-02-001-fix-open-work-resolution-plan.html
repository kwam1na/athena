fix: Close Open Work resolution gaps

Open Work stays the dense operations workspace. This plan adds source-owned resolution, type-specific row actions, and focused validation for the work types that can currently strand operators or leave stale rows behind.

Type: fix Status: active Date: 2026-07-02 Worktree: .worktrees/codex/open-work-resolution-gaps Source: Markdown plan

Direction

Keep

Existing route, operations shell, queue density, filters, pagination, row shell, and restrained operator copy.

Improve

Per-type owner/action labels, destination links, metadata, focus states, and stable numeric facts inside existing rows.

Avoid

No generic resolve-all button, no card-heavy redesign, no client-side approval authority, and no raw backend labels or metadata in visible copy.

Queue Ordering

  • Sort by priority bucket, status urgency, oldest actionable timestamp, stable source identity, then work item id.
  • Keep MAX_QUEUE_ITEMS = 100 per server lane: open, in_progress, and pending approvals.
  • Probe each lane with MAX_QUEUE_ITEMS + 1 and return overflow.workItems.open, overflow.workItems.inProgress, and overflow.approvalRequests.
  • Tests seed more than 100 rows per lane and prove the first page contains the highest-priority actionable rows.

Implementation Units

U1. Characterize Open Work projection

Lock down queue inclusion/exclusion, approval separation, source identity, duplicate policy, ordering, caps, mixed work fixtures, and existing terminal behavior in Convex and UI tests.

U2. Add type-specific row contracts

Improve titles, owner labels, action destinations, and sanitized row DTO metadata inside the current OperationsQueueView direction. Preserve density and visual language.

U3. Implement synced-sale inventory review ownership

Add the Operations-owned openWorkInventoryReviews.resolveSyncedSaleInventoryReview boundary so current inventory review work can complete, cancel, or become superseded with authz checks and audit evidence.

U4. Add daily carry-forward terminal path

Keep Opening acknowledgment as handoff evidence and add a Daily Close-owned typed completion/cancellation path requiring consumed manager proof bound to the carry-forward source.

U5. Harden source workflow terminal transitions

Cover service appointments, service cases, purchase orders, receiving, and pending checkout item review with source-owned terminal tests and fixes.

U6. Resolve approval ownership ambiguity

Keep stock/register approvals approval-owned and fail closed for service_deposit_review by suppressing it from surfaced queue data unless full proof-bound semantics are implemented.

U7. Validate integration, UI direction, and graph

Run focused tests, audit current visible rows, verify queue ordering/caps and sanitized DTOs, browser-review the Open Work route against the current workspace direction, and rebuild graph after code changes.

Highest-Risk Gaps

Synced sale inventory review

No clear terminal path was found. Resolution needs source validation, not SKU-only matching.

Daily close carry-forward

Opening acknowledgment currently leaves work open, so a separate terminal action is needed.

Service deposit review

It is fail-closed for this release: suppress it from surfaced queue data unless full proof-bound behavior is implemented.

Validation

  • Queue projection and row rendering tests cover every intentionally surfaced type.
  • POS sync and terminal currentness tests prove resolved inventory review work stops counting as current.
  • Daily Close and Opening tests separate acknowledgment evidence from terminal work status.
  • Service, procurement, receiving, pending checkout, stock adjustment, and approval tests prove source-owned terminal transitions.
  • Source identity and duplicate-policy tests prevent retries, reprojection, or reopen flows from creating duplicate current rows.
  • Queue ordering and cap/overflow tests prove actionable work remains visible or explicitly paginated.
  • Sanitized row DTO tests prevent raw metadata, proof ids, hidden financial evidence, and internal sync payloads from reaching the UI.
  • Pre-ship visible-row audit migrates, terminally updates, or suppresses any row without a resolvable source target.
  • Browser verification confirms Open Work still looks and behaves like the existing workspace, with clearer rows rather than a new design.
