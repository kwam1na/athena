Landed Change Report Handoff Gate

A digestible explanation of the new delivery report skill and the repo sensor that enforces it for large changes.

Athena workflowDelivery handoffRepo sensor2026-07-09

Why This Exists

Large branches can change enough behavior, workflow, or architecture that a PR description is not enough for a later reader to build intuition.

The delivery workflow now has an explicit report skill, and the repo now has a validation sensor that makes large branches carry that human-readable handoff artifact.

What Changed

  • Added the repo-local ce-landed-change-report skill for standalone HTML explanations with subagent-gathered context and a pass-required quiz.
  • Updated the execute skill so substantial merged deliveries generate a landed-change report as part of handoff.
  • Added landed-report:check to the PR validation provider so large source branches must include a valid docs/reports HTML artifact.
  • Stamped generated reports with data-athena-landed-change-report="v1" so sensors can distinguish intentional report artifacts from ordinary HTML.

How The Sensor Thinks

The sensor totals reportable source line changes against origin/main, excluding generated files and tests. At the default threshold, the branch must include at least one changed HTML report under docs/reports that has the report marker, subagent evidence section, and quiz form.

Operational Intuition

Compound notes teach the system durable implementation lessons. Landed-change reports teach the human reader what happened in a particular delivery. The two artifacts are intentionally separate and can both be required for the same branch.

Quiz: Pass Required

You must score at least 4 out of 5 to pass.

1. What is the main difference between a compound solution note and a landed-change report?
Compound notes preserve reusable repo knowledge. Landed-change reports explain what happened in a delivered change.
2. When does the new repo sensor require a docs/reports HTML artifact?
The check totals reportable source changes against the base ref and requires a report when the threshold is reached.
3. Which marker identifies an intentional generated landed-change report?
The renderer stamps reports with data-athena-landed-change-report="v1" so validation can recognize them.
4. Why does execute still mention post-merge report generation if the sensor runs before merge?
The gate enforces that a report artifact is present before merge, while delivery handoff can include final merge details after the merge exists.
5. What must a valid report include beyond ordinary HTML?
The sensor checks for the marker, subagent evidence, quiz heading, and quiz form id.
