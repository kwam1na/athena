Shared Demo Read Admission Rail

Delivery candidate for proving actor-aware read admission on Operations viewing, then moving demo-visible Operations, cash controls, POS, and stock-adjustment reads onto the same rail while leaving hidden reporting reads outside this migration.

Executive Summary

This change adds a public-query counterpart to the existing operation admission rail. Shared-demo reads now have explicit read definitions, adapter-based actor resolution, and store/org scope clamps before a query handler builds its read model.

Daily Operations viewing is the proof path. The exported Operations snapshot queries now use read admission with daily_operations.view, and the old shared-demo daily_operations.write bridge was removed from the view authorization helper.

After the proof path held, the same pattern moved Operations work-item and daily-close reads, cash controls register reads, POS transaction/session/register/terminal/telemetry reads, and Stock Adjustments cycle-count draft reads onto explicit read intent. A final POS register pass added the behind-the-scenes catalog, active/held session, staff roster, service catalog, and opening-guard reads that the register subscribes to before a cashier is fully signed in.

Mental Model

Before

Read surfaces could call ad hoc shared-demo helpers inside domain authorization. In Daily Operations, a read-only snapshot relied on a write capability to recognize the demo actor.

After

The exported query handler is the admission boundary. Definitions declare read intent, adapters admit the current actor, and domain helpers consume the admitted actor from context while membership-role redaction remains independent.

Failure Boundaries

  • Shared-demo actors are admitted only for definitions that allow sharedDemo: "admit".
  • Store-scoped reads deny when the requested store does not match the demo actor's server-owned store.
  • Normal users still enter through authenticated Athena user admission and store membership checks.
  • Same-store authenticated non-members are denied by the domain membership gate before session, staff, service, catalog, or daily-opening rows return.
  • Financial redaction remains membership-role based; shared-demo admission does not itself redact or unredact financial fields.
  • Public customer-channel order lookup remains unauthenticated until its storefront ownership check; operator/demo order detail uses a separate admitted query.
  • Protected external effects are denied unless shared-demo policy explicitly simulates the gateway.
  • Reporting reads are intentionally not migrated in this branch.

What Did Not Change
