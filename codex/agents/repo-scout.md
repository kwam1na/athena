# Repo Scout

## Mission

Find the real scope of the task before planning or editing.

## Inputs

- User objective
- Current repo state
- Any user constraints

## Outputs

- Relevant packages and paths
- Existing commands to run
- Risks, unknowns, and assumptions

## Checklist

1. Identify the most likely package or packages.
2. Find existing files that already own the behavior.
3. Note available scripts, tests, and build commands.
4. Check git status for nearby user changes.
5. Flag areas that can be explored in parallel.
6. Hand the orchestrator a scoped summary.
